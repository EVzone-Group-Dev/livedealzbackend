================================================================================
  DEPLOYMENT DIRECTORY
================================================================================

Server Path: /var/www/livedealzbackend

This is where the application will be deployed on your DigitalOcean Droplet.

================================================================================
  REQUIRED GITHUB SECRETS
================================================================================

You MUST configure these secrets in GitHub:

1. DO_HOST
   - Your Droplet's IP address or hostname
   - Example: 123.45.67.89

2. DO_USERNAME
   - SSH username for your Droplet
   - Example: ubuntu or root

3. DO_SSH_KEY
   - Private SSH key for authentication
   - Generate with: ssh-keygen -t ed25519 -f deploy_key
   - Add public key to ~/.ssh/authorized_keys on Droplet

================================================================================
  DEPLOYMENT PROCESS
================================================================================

1. Developer pushes code to main branch
   ↓
2. GitHub Actions triggers workflow
   ↓
3. Test Job:
   - Checks out code
   - Sets up Node.js 20
   - Installs dependencies
   - Runs all tests (npm run test)
   ↓
4. If tests pass → Deploy Job:
   - Connects to Droplet via SSH
   - Backs up config files
   - Pulls latest code
   - Installs dependencies (npm ci)
   - Generates Prisma client
   - Builds application (npm run build)
   - Restarts with PM2
   ↓
5. Deployment complete! 🚀

================================================================================
  QUICK START
================================================================================

1. Set up GitHub Secrets:
   - Repository Settings → Secrets and variables → Actions
   - Add DO_HOST, DO_USERNAME, DO_SSH_KEY

2. Configure Droplet:
   - Install Node.js 20 and PM2
   - Create /var/www/livedealzbackend directory
   - Set up SSH key for GitHub Actions

3. Create Environment:
   - Create .env file on server
   - Copy ecosystem.config.example.js to ecosystem.config.js

4. Test Deployment:
   - Push a commit to main branch
   - Monitor GitHub Actions
   - Verify with pm2 status

================================================================================
  TROUBLESHOOTING
================================================================================

If deployment fails:
1. Check GitHub Actions logs
2. Verify SSH key configuration
3. Ensure Droplet is accessible
4. Check disk space on Droplet
5. Review PM2 logs: pm2 logs livedeals-api

If tests fail:
1. Check Actions tab for error details
2. Run tests locally: npm run test
3. Verify test database connection

If app won't start:
1. Check PM2 logs: pm2 logs livedeals-api
2. Verify .env file exists
3. Check port availability: lsof -i :3000
4. Verify Prisma can connect to database
